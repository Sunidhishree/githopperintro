RepoScan AI — Backend Pipeline

Full end-to-end flow: from GitHub URL → Bedrock AI scans → Repo Health Card

Stage 1 · Frontend
1React Frontend (S3 / Amplify)
User pastes a public GitHub repo URL and clicks Scan.
Frontend sends POST /scan with { repoUrl: "..." } to API Gateway.
React.js Tailwind CSS AWS Amplify
HTTPS POST /scan
Stage 2 · API Layer
2Amazon API Gateway
Receives the POST request. Validates payload, enforces rate limits, handles CORS for the React origin. Triggers Lambda #1 synchronously.
REST API CORS Rate limiting
Invokes Lambda
Stage 3 · Lambda #1
3github-fetcher Lambda (Node.js)
Calls GitHub REST API with the repo URL.
Fetches: full file tree, raw source file contents, package.json / requirements.txt, IaC files (Terraform, CloudFormation). Returns a structured file map to the next Lambda.
GitHub REST API AWS Lambda
Raw file map
Stage 4 · Lambda #2
4code-processor Lambda (Python)
Prepares code for AI analysis:
• Chunking — splits large files to fit Bedrock's context window
• Dep parsing — extracts library versions from manifests
• IaC detection — identifies S3, IAM, security group configs
Outputs two payloads: one for security scan, one for debt scan.
AWS Lambda Context chunking Dep parsing
Two structured payloads
Stage 5a · AI Scan
5Bedrock — Security Scan
Claude 3 Sonnet via InvokeModel.
Detects: hardcoded secrets & API keys, insecure/outdated dependencies, IaC misconfigs (S3 public, IAM *, open SGs), injection vulns, exposed endpoints.
Returns structured JSON findings.
Claude 3 Sonnet JSON mode
Stage 5b · AI Scan
5Bedrock — Debt Scan
Claude 3 Sonnet via InvokeModel.
Detects: high cyclomatic complexity functions, duplicate/copy-paste logic, missing test coverage, tightly coupled modules, abandoned feature flags.
Returns structured JSON findings.
Claude 3 Sonnet JSON mode
Merged findings JSON
Stage 6 · Lambda #3
6scorer Lambda (Python)
Aggregates all findings from both Bedrock calls.
Runs severity × effort scoring on each finding.
Computes: Security Health score, Debt Health score, Overall Repo Health Score (0–100).
Builds prioritised action plan — quick wins and critical blockers surfaced first.
Scoring logic AWS Lambda
Scored findings
Stage 7 · AI Report
7Bedrock — Report Formatter
Final Claude 3 Sonnet call.
Takes scored findings JSON and generates the full Repo Health Card: plain-English explanations, fix suggestions with code snippets, estimated fix times. Output is a clean JSON blob ready for the React frontend.
Claude 3 Sonnet Plain-English output
Health Card JSON
Stage 8a · Storage
8Amazon S3
Report JSON saved as an artifact in S3. Generates a shareable URL so teams can revisit past scans and track score improvement over time.
S3 artifact Shareable link
Stage 8b · Display
8React UI — Health Card
Response rendered in the React frontend: score gauge, security findings, debt findings, prioritised action plan, and quick wins section.
Health Card Export option
Cross-cutting AWS services (active throughout)
CloudWatch
Logs all Lambda invocations. Real-time error + latency metrics during demo.
IAM Roles
Fine-grained per-Lambda roles. Only the AI Lambda has bedrock:InvokeModel. S3 write scoped to output bucket only.
AWS Amplify
CI/CD for React app. Provides HTTPS endpoint. Connects to S3 for static assets.
